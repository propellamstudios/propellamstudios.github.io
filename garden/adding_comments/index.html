Adding comments with Giscus

Thoughts on different commenting systems for static sites

This blog is created using hugo, which is a open source framework used to generate static sites. Static sites are websites where all the content is ‘fixed’ and served up to the user as is; the site does not change response to user interaction. Most - maybe all? - of the biggest and most popular sites are not static sites, but are instead dynamic sites, since they have to respond to user interaction to do whatever it is you’re trying to do. How would Google work if it couldn’t serve you a list of results to your search query? What would be the point of Amazon if you couldn’t add items to your shopping cart, then checkout and order them? But that responsiveness and dynamic nature comes at the cost of a lot more backend infrastructure; it usually involves setting up a database, and also writing client-side (the user) or server-side scripting which governs how the pages that get served to the user change.

This type of thing is excessive for small blogs like this one, which is why static sites frameworks like Hugo or Jekyll are quite popular for people looking to create blogs. Static sites are generally much more lightweight and easier to both create and maintain than dynamic sites, and it’s easier to find free options to run/host static sites compared to dynamic sites.

However, one piece of functionality that blogs do often have or want that is difficult to natively implement in static sites is commenting. If you think about it, commenting is a dynamic interaction; someone leaves a comment on a post, and then the website changes in order to display the post. So how can a static site have commenting, when the entire idea is that it has fixed content?

Pondering this question sent me down a rabbit hole of looking at a whole array of options, because it turns out that quite a lot of other people have asked that question before, and come up with a lot of different solutions. One of the most popular solutions is Disqus, which is a commenting plugin that can be added to blogs or other types of websites. Disqus has a lot of nice features; it can allow for anonymous commenting, or require people include their name and email address in order to submit a comment. You can also add on other tools to Disqus in order to prevent spam or bots, e.g. reCaptcha. But there are substantial downsides, too, at least in my opinion: the free tier inserts an array of ads at the bottom of the page - you know, random photos with the captions like, “Lose weight with this one weird trick!”, or, “Make money while you sleep!” Or advertising products that look useless, or blasting headlines about ‘celebrities’ that you’ve never heard of before. You know, that type of thing. The other thing is that the text of comments aren’t really owned by either the site owner or the commenter, but rather by Disqus. Both of those drawbacks pretty much ruled out Disqus for me.

There are a lot of other paid commenting systems out there, e.g. Commento or Commentbox (although I’m not sure Commentbox is still actively maintained), but it seems like a little much to pay for a commenting system when I’m definitely not making any money off this blog (which is okay! I don’t ever intend to insert ads etc.), and when I’m not sure if there are really going to be comments anyways.

Reading through a bunch of other folks’ blog posts about this ended up leading to a couple other options that I seriously considered: Isso and Staticman. I read a lot about both systems, but they both require using a remote server to store and serve comments. For both systems, it appears that there is very thorough documentation about how to do that, and I probably could have figured it out if I’d been willing to sink more time and effort into it, but again, this seemed to be an excess of effort; I’m not particularly interested in learning backend web development, I’d rather spend that time working on other things. But, if you’re reading this and trying to figure out what commenting system to use for your static site, I think that Isso and Staticman were the two most promising frameworks that I came across.

After all the reading I ended up doing, I ended up implementing Giscus, which is free open source software (the repo is here). It was actually very simple to add to the Hugo theme I’m using for this blog (the digital garden theme, created by Paul Martins) - I was honestly quite surprised and almost a little suspicious as to how easy it was! These two blog posts - Switching from Disqus to Giscus and Giscus: The New Commenting Engine for My Website were both extremely useful.

Giscus is a commenting system that is based off of Github Discussions. What git and github are is an entirely different discussion/blog post - and I might actually write a post about it at some point - but in a very, very abbreviated summary, git is a type of version control software that allows users to keep tracked ‘versions’ of their projects; each project is contained in a repository, which can be saved locally and also on a remote server. Github is a website (and a company that became a subsidiary of Microsoft in 2018) that hosts repositories; that is, it’s probably the most popular option for that remote server where people save their repos. Github discussions is a feature on Github that allows Github users to have discussions linked to specific repositories. The creater of Giscus wrote the Giscus app to re-purpose the Github discussions feature to provide comments for static sites, and overall, it works pretty well; I’ve commented on a few other people’s blogs that use Giscus, and it was a pretty smooth system.

The one big drawback to using Giscus is that in order to use it to leave comments, you have to have a Github account. It’s free and quite simple to sign up for a Github account, but unless you’re a software developer, it may be unlikely for you to have a Github account already; even though git is not specific to software development - and in fact, I think that git would be broadly useful for lots of different types of digital work, although it apparently doesn’t work quite as smoothly for binary files or very large files - what I’ve found in practice is that unless someone is in tech, either professionally or as a hobby, it’s unlikely for them to have heard of git or github. The other blogs that I’ve found that use Giscus as a commenting tool are typically tech blogs, where most of the audience probably would have Github accounts. For this blog?
